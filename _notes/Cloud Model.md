# 클라우드 모델이란
[[Cloud|클라우드]] 서비스가 컴퓨팅 자원을 제공하는 형태를 개념적으로 정의한 것
## 접근성 측면에서의 클라우드 모델
**[[Public Cloud]]** - CSP가 구축한 컴퓨팅 리소스를 돈을 지불하고 이용
- 서비스 사용량에 기반한 지불
- 유지 관리의 부담 완화
- 높은 안정성
- 높은 확장성
**[[Private Cloud]]** - 자체적인 데이터센터 내부에 [[Cloud|클라우드]] 환경을 구축 및 사용
- 커스텀 가능한 클라우드 환경
- 보안 수준 강화
- 높은 확장성
**[[Hybrid Cloud]]** - [[Private Cloud]]와 [[Public Cloud]]를 혼합 사용
- 중요 데이터는 Private에 관리 가능
- Public Cloud와 Private Cloud의 리소스 비율을 제어하여 효율적인 비용 관리
- 클라우드 전환의 용이성

### [[Infrastructure as a Service]]
- OS 수준부터 어플리케이션 수준까지 관리
- 물리적 리소스를 가상화
### [[Platform as a Service]]
- 데이터 수준부터 어플리케이션 수준까지 관리
### [[Software as a Service]]
- 아무것도 관리하지 않아도 됨
